Complement C++ libraries

What is this

Complement is a set of C++ libraries. This libraries provide multiplatform C++ framework for multithreaded programming and communication.

Brief Description

xmt
Portable programming interface to build MultiThreaded (MT) applications or libraries. Hide platform-specific system calls that used for MT-programming.
sockios
Generic interface for interprocess communication via sockets. Sockios programming interface unified with C++ standard (ISO/IEC 14882) IO streams interface.
EDS
EDS is Event Driven System. Framework for asynchronous events delivery and processing. Here a bit more than only callback interface. Object that process some event has a so-called 'state' (really stack of states), so event processors may vary while object change state (as specified in compile time). Communication with events in EDS framework possible as within single process, as between different processes (wold-wide too), over TCP or UDP (framework provided by libsockios). In contrast to similar frameworks, to process EDS-designed code you need only C++ compiler.

Present State of Complement Project

Active development for Unix-like systems (Solaris and Linux, I have no access to HP-UX now) with gcc 3.1.1/STLport 5.x. Some time ago this libraries was working on Windows (95, 98, NT) with VC++ 5 SP3 (client part, connections processing server never work on MS Windows), HP-UX 11.00 (HP aCC A.03.13) and Solaris (SunSoft CC 5.2).
