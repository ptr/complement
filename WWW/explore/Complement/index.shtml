Complement C++ libraries

What is this

Complement is a set of C++ libraries. This libraries provide multiplatform C++ framework for multithreaded programming and communication.

Brief Description

xmt
Portable programming interface to build MultiThreaded (MT) applications or libraries. Hide platform-specific system calls that used for MT-programming.
sockios
Generic interface for interprocess communication via sockets. Sockios programming interface unified with C++ standard (ISO/IEC 14882) IO streams interface.
EDS
EDS is Event Driven System. Framework for asynchronous events delivery and processing. Here a bit more than only callback interface. Object that process some event has a so-called 'state' (really stack of states), so event processors may vary while object change state (as specified in compile time). Communication with events in EDS framework possible as within single process, as between different processes (wold-wide too), over TCP or UDP (framework provided by libsockios). In contrast to similar frameworks, to process EDS-designed code you need only C++ compiler.

Other requirements

  • GNU make v. 3.79.1 (for Unix-like platforms);
  • Good C++ compiler; I hope now it is not problem: you can use GNU gcc 2.95.3 or 3.1.1 (other gcc versions may work too, but in particular I am not recommend to use gcc before 2.95.3, and early 3.x releases; 2.96, that shipout with RedHat, IMHO isn't good choice too);
  • STLport---STL implementation. Very efficient, very portable STL implementation. Instead of have troubles with different (and sometimes ugly) STL implementations that come with C++ compilers, I use STLport.
  • Python interpreter to make more-or-less automatic tests.

Present State of Complement Project

Active development for Unix-like systems (Solaris and Linux, I have no access to HP-UX now) with gcc 3.1.1/STLport 5.x. Some time ago this libraries was working on Windows (95, 98, NT) with VC++ 5 SP3 (client part, connections processing server never work on MS Windows), HP-UX 11.00 (HP aCC A.03.13) and Solaris (SunSoft CC 5.2).

CVS access

You can read directives for CVS access here or below on this page.

Anonymous CVS Access

This project's SourceForge.net CVS repository can be checked out through anonymous (pserver) CVS with the following instruction set. When prompted for a password for anonymous, simply press the Enter key. 

cvs -d:pserver:anonymous@cvs.complement.sourceforge.net:/cvsroot/complement login 

cvs -z3 -d:pserver:anonymous@cvs.complement.sourceforge.net:/cvsroot/complement co complement

Updates from within the module's directory do not need the -d parameter.

Developer CVS Access via SSH

Only project developers can access the CVS tree via this method. SSH (either SSH1 or SSH2) must be installed on your client machine. Substitute developername with the proper values. Name of module is 'complement' here. Enter your site password when prompted. 

export CVS_RSH=ssh

cvs -z3 -d:ext:developername@cvs.complement.sourceforge.net:/cvsroot/complement co complement
